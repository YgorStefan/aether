Design — Seção 1: Arquitetura Geral

Visão do sistema completo — três camadas independentes conectadas por interfaces bem definidas

Aether OS — System Architecture
CAMADA 1 — FRONTEND
📦 Next.js 15 (App Router)
· Bento Grid Dashboard
· Chat Interface (SSE)
· React Flow Graph View
· HITL Approval Dialog
· Auth (Supabase Client)
CAMADA 2 — BACKEND
🐍 FastAPI (Python)
· Agent Engine (LangGraph)
· Supervisor + Workers
· Skill Registry
· Budget Controller
· MCP Server
CAMADA 3 — DATA
🗄️ Supabase
· Auth + JWT
· PostgreSQL (runs, users)
· pgvector (memória RAG)
· RLS (isolamento por user)
· LangSmith (traces)
— Fluxo de uma request —
User input
→
Next.js
→
FastAPI
(JWT validated)
→
LangGraph
Supervisor
→
Workers +
Skills
→
SSE stream
→ UI
SERVIÇOS EXTERNOS
🔍 Tavily (WebSearch) 🐍 E2B (CodeInterpreter sandbox) 🧠 Gemini Flash (LLM) 📊 LangSmith (traces)

Essa arquitetura usa três fronteiras claras: UI não conhece o banco, FastAPI não conhece o Supabase Client, agentes não conhecem HTTP. Cada camada é testável e substituível de forma independente.
